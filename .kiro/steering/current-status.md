# Current Status Steering

## Rules

1. **At the start of every new session**, read `CURRENT_STATUS.md` in the repository root to understand the current state of the project before taking any action.

2. **Use CURRENT_STATUS.md as the primary context source** for understanding:
   - What has been built
   - What is in progress
   - Key technical decisions
   - Next steps and priorities

3. **Update CURRENT_STATUS.md at the end of each session** (or when significant progress is made) with:
   - Any new features completed
   - Changes to "In Progress" section
   - New technical decisions made
   - Updated "Next Steps" if priorities change
   - Updated "Last Updated" timestamp

4. **Reference file:** #[[file:CURRENT_STATUS.md]]

5. **Workflow:**
   - Start of session → Read `CURRENT_STATUS.md` → Understand context → Begin work
   - End of session → Update `CURRENT_STATUS.md` → Commit and push

6. **Keep it concise:** CURRENT_STATUS.md should be a quick-reference document, not a full history. For detailed history, refer to `chat-log/chat-log.md`.
